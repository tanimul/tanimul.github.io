The difference between lateinit vs lazy keyword in kotlin

lateinit- 1. lateinit promises the compiler that variables will be initialized before use, but throws an UninitializedPropertyAccessException if they aren't. 2. lateinit can't be compiled to final fields, so it can only be applied to var keyword. So immutability cannot be guaranteed. 3. Can be only used with a non-nullable variable. 4. It will not allocate memory until initialized. lazy- 1. Your variable will not be initialized unless you use that variable in your code. 2. It is a read-only property as it can only be used with the val keyword. 3. It only creates the object when we access it for the very first time and then in the subsequent access, it returns the same object.
